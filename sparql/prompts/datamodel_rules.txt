DATAMODELREGELS RCE CEO

Doel:
Deze regels helpen de LLM om geldige SPARQL te maken voor het RCE CHO endpoint.
Gebruik deze regels boven algemene aannames.

PREFIXEN

Gebruik deze prefixen:

PREFIX ceo: <https://linkeddata.cultureelerfgoed.nl/def/ceo#>
PREFIX graph: <https://linkeddata.cultureelerfgoed.nl/graph/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

Gebruik nooit:
- ceosp:
- ceox:

ALGEMENE REGELS

- Genereer alleen SPARQL.
- Gebruik geen uitleg buiten de SPARQL-query.
- Verzin geen classes.
- Verzin geen properties.
- Gebruik geen meervoud als classnaam.
- Gebruik alleen termen uit de CEO of uit deze bewezen patronen.
- Gebruik bij lijstvragen LIMIT, tenzij de gebruiker expliciet alle resultaten vraagt.
- Gebruik COUNT alleen in tellingmodus.
- Gebruik DISTINCT bij tellingen met joins.
- Gebruik OPTIONAL voor naam, omschrijving, adres, type, functie en geometrie.
- Gebruik geen SELECT *.

BASISREGISTRATIE EN LOCATIE

Provincie staat niet direct op een cultuurhistorisch object.
Provincie loopt via ceo:heeftBasisregistratieRelatie.

Correct patroon:

?cho ceo:heeftBasisregistratieRelatie ?basisrelatie .
?basisrelatie ceo:heeftProvincie <PROVINCIE_URI> .

Gebruik nooit:

?cho ceo:heeftProvincie <PROVINCIE_URI> .

Gemeente loopt ook via basisregistratie:

?cho ceo:heeftBasisregistratieRelatie ?basisrelatie .
?basisrelatie ceo:heeftGemeente ?gemeente .

Adres en woonplaats lopen via BAGRelatie:

?cho ceo:heeftBasisregistratieRelatie ?basisrelatie .
?basisrelatie ceo:heeftBAGRelatie ?bag .
OPTIONAL { ?bag ceo:woonplaatsnaam ?woonplaats . }
OPTIONAL { ?bag ceo:volledigAdres ?adres . }

Zoeken op woonplaatsnaam:

?cho ceo:heeftBasisregistratieRelatie ?basisrelatie .
?basisrelatie ceo:heeftBAGRelatie ?bag .
?bag ceo:woonplaatsnaam ?woonplaats .
FILTER(LCASE(STR(?woonplaats)) = "zeist")

NAAM

Naam staat op een ceo:Naam-node.

Correct patroon:

OPTIONAL {
  ?cho ceo:heeftNaam ?naamNode .
  ?naamNode ceo:naam ?naam .
}

OMSCHRIJVING

Omschrijving staat op een ceo:Omschrijving-node.

Correct patroon:

OPTIONAL {
  ?cho ceo:heeftOmschrijving ?omschrijvingNode .
  ?omschrijvingNode ceo:omschrijving ?omschrijving .
}

Omschrijving is vrije tekst.
Gebruik omschrijving alleen voor tonen of als laatste redmiddel voor zoeken.
Als een resultaat alleen via omschrijving matcht, voeg toe:

BIND("omschrijving (onzeker)" AS ?bron)

LOCATIEAANDUIDING

Locatieaanduiding staat op een ceo:LocatieAanduiding-node.

Correct patroon:

OPTIONAL {
  ?cho ceo:heeftLocatieAanduiding ?locatie .
  OPTIONAL { ?locatie ceo:locatienaam ?locatienaam . }
  OPTIONAL { ?locatie ceo:locatieomschrijving ?locatieomschrijving . }
}

AACTOREN EN ROLLEN

Gebruik geen ceo:architect.
Architectinformatie loopt via ceo:Gebeurtenis en ceo:ActorEnRol.

CEO-patroon:

?cho ceo:heeftGebeurtenis ?gebeurtenis .
?gebeurtenis ceo:heeftActorEnRol ?actorEnRol .
?actorEnRol ceo:heeftActor ?actor .
?actorEnRol ceo:heeftRol ?rol .

Volgens de CEO hebben ceo:heeftActor en ceo:heeftRol range skos:Concept,
maar in de huidige instance-data kunnen deze waarden literals zijn.
Gebruik daarom STR() bij filters.

Voor vragen met:
- architect
- ontwerper
- bouwmeester
- bouwkundige
- constructeur

filter op ceo:heeftRol:

FILTER(CONTAINS(LCASE(STR(?rol)), "architect"))

Voor vragen naar een specifieke persoon of organisatie als actor,
filter op ceo:heeftActor:

FILTER(CONTAINS(LCASE(STR(?actor)), "kramer"))

Vraagpatroon:
"Van welk rijksmonument is Kramer, Hendrik de architect?"

Correct querypatroon:

SELECT DISTINCT ?cho ?rijksmonumentnummer ?naam ?actor ?rol
WHERE {
  ?cho a ceo:Rijksmonument .

  ?cho ceo:heeftGebeurtenis ?gebeurtenis .
  ?gebeurtenis ceo:heeftActorEnRol ?actorEnRol .
  ?actorEnRol ceo:heeftActor ?actor .
  ?actorEnRol ceo:heeftRol ?rol .

  FILTER(CONTAINS(LCASE(STR(?actor)), "kramer"))
  FILTER(CONTAINS(LCASE(STR(?rol)), "architect"))

  OPTIONAL { ?cho ceo:rijksmonumentnummer ?rijksmonumentnummer . }

  OPTIONAL {
    ?cho ceo:heeftNaam ?naamNode .
    ?naamNode ceo:naam ?naam .
  }
}
LIMIT 50

MONUMENTAARD

ceo:heeftMonumentAard geeft de aard van een rijksmonument aan.
De waarde is een skos:Concept.

De relevante skos:prefLabel waarden zijn:
- "onroerend gebouwd"
- "archeologisch"

Archeologische rijksmonumenten:

?cho ceo:heeftMonumentAard ?monumentaard .
?monumentaard skos:prefLabel ?monumentaardLabel .
FILTER(
  LCASE(STR(?monumentaardLabel)) = "archeologisch"
)

Gebouwde rijksmonumenten:

?cho ceo:heeftMonumentAard ?monumentaard .
?monumentaard skos:prefLabel ?monumentaardLabel .
FILTER(
  LCASE(STR(?monumentaardLabel)) = "onroerend gebouwd"
)

Gebruik voor "archeologische rijksmonumenten" altijd ceo:heeftMonumentAard.
Gebruik niet ceo:ArcheologischTerrein of ceo:ArcheologischComplex, tenzij de gebruiker expliciet naar terreinen of complexen vraagt.

RIJKSMONUMENTEN

Class:

?cho a ceo:Rijksmonument .

Bewezen directe properties op ceo:Rijksmonument:
- ceo:cultuurhistorischObjectnummer
- ceo:rijksmonumentnummer
- ceo:datumInschrijvingInMonumentenregister
- ceo:registratiedatum
- ceo:heeftBasisregistratieRelatie
- ceo:heeftGeometrie
- ceo:heeftJuridischeStatus
- ceo:heeftKennisregistratie
- ceo:heeftLocatieAanduiding
- ceo:heeftMonumentAard
- ceo:heeftNaam
- ceo:heeftOmschrijving
- ceo:heeftOorspronkelijkeFunctie
- ceo:isHoofdobjectVanComplex
- ceo:isOnderdeelVanComplex


Rijksmonumentnummer:

OPTIONAL { ?cho ceo:rijksmonumentnummer ?rijksmonumentnummer . }

Cultuurhistorisch objectnummer:

OPTIONAL { ?cho ceo:cultuurhistorischObjectnummer ?choNummer . }

Rijksmonumenten in provincie:

SELECT ?cho ?rijksmonumentnummer ?naam ?woonplaats ?adres
WHERE {
  ?cho a ceo:Rijksmonument .

  ?cho ceo:heeftBasisregistratieRelatie ?basisrelatie .
  ?basisrelatie ceo:heeftProvincie <http://standaarden.overheid.nl/owms/terms/Utrecht_(provincie)> .

  OPTIONAL { ?cho ceo:rijksmonumentnummer ?rijksmonumentnummer . }

  OPTIONAL {
    ?cho ceo:heeftNaam ?naamNode .
    ?naamNode ceo:naam ?naam .
  }

  OPTIONAL {
    ?basisrelatie ceo:heeftBAGRelatie ?bag .
    OPTIONAL { ?bag ceo:woonplaatsnaam ?woonplaats . }
    OPTIONAL { ?bag ceo:volledigAdres ?adres . }
  }
}
LIMIT 50

Archeologische rijksmonumenten in provincie:

SELECT (COUNT(DISTINCT ?cho) AS ?aantal)
WHERE {
  ?cho a ceo:Rijksmonument .

  ?cho ceo:heeftBasisregistratieRelatie ?basisrelatie .
  ?basisrelatie ceo:heeftProvincie <http://standaarden.overheid.nl/owms/terms/Utrecht_(provincie)> .

  ?cho ceo:heeftMonumentAard ?monumentaard .
  ?monumentaard skos:prefLabel ?monumentaardLabel .

  FILTER(
      LCASE(STR(?monumentaardLabel)) = "archeologisch"
  )
}

COMPLEXEN

Class:

?complex a ceo:Complex .

Bewezen directe properties op ceo:Complex:
- ceo:complexnummer
- ceo:cultuurhistorischObjectnummer
- ceo:heeftHoofdobject
- ceo:heeftKennisregistratie
- ceo:heeftNaam
- ceo:heeftRijksmonument
- ceo:registratiedatum

Complexen hebben geen bewezen directe locatie.
Gebruik gekoppelde rijksmonumenten of hoofdobject voor locatievragen.

Complexen in provincie:

SELECT DISTINCT ?complex ?complexnummer ?naam
WHERE {
  ?complex a ceo:Complex .
  ?complex ceo:heeftRijksmonument ?rijksmonument .

  ?rijksmonument ceo:heeftBasisregistratieRelatie ?basisrelatie .
  ?basisrelatie ceo:heeftProvincie <http://standaarden.overheid.nl/owms/terms/Utrecht_(provincie)> .

  OPTIONAL { ?complex ceo:complexnummer ?complexnummer . }

  OPTIONAL {
    ?complex ceo:heeftNaam ?naamNode .
    ?naamNode ceo:naam ?naam .
  }
}
LIMIT 50

ARCHEOLOGISCHE COMPLEXEN

Class:

?complex a ceo:ArcheologischComplex .

Gebruik nooit:

?complex a ceo:ArcheologischeComplexen .

Bewezen directe properties op ceo:ArcheologischComplex:
- ceo:cultuurhistorischObjectnummer
- ceo:heeftGebeurtenis
- ceo:heeftKennisregistratie
- ceo:heeftType
- ceo:ligtInObject
- ceo:registratiedatum

Een ceo:ArcheologischComplex heeft in de instance-data geen bewezen directe basisregistratie.
Gebruik voor locatie/provincie het bovenliggende onderzoeksgebied via ceo:ligtInObject.

Correct locatiepad:

?complex ceo:ligtInObject ?onderzoeksgebied .
?onderzoeksgebied a ceo:ArcheologischOnderzoeksgebied .
?onderzoeksgebied ceo:heeftBasisregistratieRelatie ?basisrelatie .
?basisrelatie ceo:heeftProvincie <PROVINCIE_URI> .

Gebruik niet:

?complex ceo:heeftBasisregistratieRelatie ?basisrelatie .
?complex ceo:heeftProvincie <PROVINCIE_URI> .

FUNCTIES

Functie-node:

?functie a ceo:Functie .

Bewezen properties op ceo:Functie:
- rdf:type ceo:Functie
- rdf:type ceo:Kennisregistratie
- ceo:formeelStandpunt
- ceo:heeftBetrekkingOp
- ceo:heeftFunctieNaam
- ceo:hoofdfunctie
- ceo:kennisregistratienummer
- ceo:registratiedatum

Vanaf rijksmonument naar oorspronkelijke functie:

?cho ceo:heeftOorspronkelijkeFunctie ?functie .
?functie ceo:heeftFunctieNaam ?functieConcept .
OPTIONAL { ?functieConcept skos:prefLabel ?functieLabel . }

Via kennisregistratie:

?functie a ceo:Functie ;
         ceo:heeftBetrekkingOp ?cho ;
         ceo:heeftFunctieNaam ?functieConcept .
OPTIONAL { ?functieConcept skos:prefLabel ?functieLabel . }

Zoeken naar kerk via functie:

?cho ceo:heeftOorspronkelijkeFunctie ?functie .
?functie ceo:heeftFunctieNaam ?functieConcept .
?functieConcept skos:prefLabel ?functieLabel .
FILTER(CONTAINS(LCASE(STR(?functieLabel)), "kerk"))

TYPE

Type-node:

?type a ceo:Type .

Algemeen patroon:

?cho ceo:heeftType ?type .
?type ceo:heeftTypeNaam ?typeConcept .
OPTIONAL { ?typeConcept skos:prefLabel ?typeLabel . }

Zoeken op type, bijvoorbeeld kasteel:

?cho ceo:heeftType ?type .
?type ceo:heeftTypeNaam ?typeConcept .
?typeConcept skos:prefLabel ?typeLabel .
FILTER(CONTAINS(LCASE(STR(?typeLabel)), "kasteel"))

TELLINGEN

Gebruik DISTINCT bij objecttellingen:

SELECT (COUNT(DISTINCT ?cho) AS ?aantal)
WHERE {
  ?cho a ceo:Rijksmonument .
}

Voor objecten met aantalsproperty’s:

Vondsten:

SELECT (SUM(?aantalVondsten) AS ?aantal)
WHERE {
  ?vondst a ceo:Vondsten .
  ?vondst ceo:aantalVondsten ?aantalVondsten .
}

Grondsporen:

SELECT (SUM(?aantalGrondsporen) AS ?aantal)
WHERE {
  ?grondspoor a ceo:Grondsporen .
  ?grondspoor ceo:aantalGrondsporen ?aantalGrondsporen .
}

OUTPUTREGELS

- Gebruik geen SELECT *.
- Geef altijd de URI terug van het hoofdobject.
- Gebruik duidelijke variabelen.
- Voeg relevante nummer-velden toe:
  - ?rijksmonumentnummer
  - ?complexnummer
  - ?archis2Monumentnummer
  - ?choNummer
- Voeg waar mogelijk ?naam, ?woonplaats, ?adres of ?locatienaam toe.
- Gebruik LIMIT 50 als geen ander aantal is gevraagd.

GEOMETRIE

Gebruik GeoSPARQL voor geometrie.

Prefix:

PREFIX geo: <http://www.opengis.net/ont/geosparql#>

Correct patroon:

?cho ceo:heeftGeometrie/geo:asWKT ?wkt .

Gebruik bij kaartvragen altijd variabele ?wkt.

Voor kaartvragen of vragen met woorden zoals:
- kaart
- geometrie
- locatie op kaart
- toon op kaart
- waar liggen

voeg toe:

OPTIONAL {
  ?cho ceo:heeftGeometrie/geo:asWKT ?wkt .
}

Of verplicht als de gebruiker expliciet een kaart wil:

?cho ceo:heeftGeometrie/geo:asWKT ?wkt .
